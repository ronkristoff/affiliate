# Design Context — salig-affiliate

## Users

- **SaaS Owner ("Alex")** — Running a subscription SaaS business in PH/SEA; needs to launch, manage, and track affiliate programs with minimal friction; values accuracy and automation
- **Affiliate ("Jamie")** — Content creators, freelancers, digital marketers; non-technical; needs clear, accessible dashboards; values trust and transparency
- **Platform Admin** — Internal operations team; needs efficient tenant management tools

## Brand Personality

**Voice**: Modern Professional — confident, precise, community-rooted
**3-word personality**: Trustworthy, Data-driven, Approachable
**Emotional goals**: Confidence-inspiring without being stiff; FinTech-grade precision

## Aesthetic Direction

- **Visual tone**: Command-center dashboard — clean, data-focused, modern tooling feel
- **Theme**: Light mode primary, dark mode available
- **Primary**: `#1c2260` (deep navy), **Secondary**: `#1fb5a5` (teal/mint)
- **Typography**: Poppins (body), Passion One (display headings)
- **Border radius**: 12px (0.75rem) default
- **Status colors**: success (emerald), warning (amber), danger (red), info (blue)
- **Anti-references**: Corporate stiffness, generic SaaS blue, cluttered dashboards

## Design Principles

1. **Trust through precision** — Financial accuracy is paramount; UI must convey reliability down to the pixel
2. **Progressive disclosure** — Show complexity only when needed; keep onboarding friction low
3. **Brand system consistency** — Gradient accent bars, badge components, and CSS variable tokens are the single source of truth; never introduce one-off color values
4. **Mobile-first for affiliates** — Dashboard must be responsive; affiliates may check commissions on mobile
5. **Clear status communication** — Commission states, payout status, fraud flags must be instantly understandable
