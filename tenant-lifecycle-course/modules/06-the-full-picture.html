The engineering behind the curtain

This system makes several deliberate architectural choices that are worth understanding — they come up in almost every production SaaS:

🔒

Tenant Isolation

Every query starts with tenantId. It's impossible for one company to see another's data — not through a bug, not through a hack. This is called multi-tenancy and it's the foundation of every SaaS.

⚡

Denormalized Counters

Stats are pre-computed, not calculated on demand. This trades write complexity (every mutation must update counters) for read speed (dashboards load instantly). The counters are updated in the same transaction as the data change, so they never drift.

🔗

Atomic Mutations

Payout batching either succeeds completely or fails completely. No partial payouts. This uses Convex's transaction guarantee — the database either commits all changes or rolls back to the state before the mutation started.

📝

Audit Everything

Every significant action creates an audit log: tenant created, user removed, commission approved, payout generated, domain changed. This is non-negotiable for a financial system — regulators require it, and it's invaluable for debugging.

🛡️

Layered Fraud Detection

Self-referral checks (email + IP match), click deduplication (same person clicking 100 times), payment status validation (reject "pending" and "failed" payments), and auto-approval thresholds (manual review for large commissions).

🎯

Triple Attribution

Cookie → Coupon → Email lead matching. If the first method fails, try the next. Coupon beats cookie (explicit > implicit). If all fail, record as organic — never lose a sale's data.

When things break — and how to find out why

Here are the most common failure points in the tenant lifecycle — and where to look:

Scenario

"Alex says an affiliate referred a customer who bought a $500 subscription, but no commission appeared. Where do you investigate?"

1

rawWebhooks table
Did the webhook arrive? Check for the event ID. Status: "processed" or "error"?

→
2

Attribution
Was there a cookie? Coupon code? Lead match? Check the "No attribution" log message.

→
3

Affiliate status
Is the affiliate still "active"? Suspended affiliates don't earn commissions.

→
4

Payment status
Was the webhook payment status "paid" or "completed"? "pending" payments are rejected.

→
5

Campaign status
Is the campaign "active"? Paused or archived campaigns block commissions.

ℹ️
The audit log is your best friend

Almost every failure leaves a trail in the audit logs. The commission engine logs rejection reasons, the webhook handler logs processing errors, and the attribution system logs resolution paths. Start there.
